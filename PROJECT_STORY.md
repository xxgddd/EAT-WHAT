# 🕵️‍♂️ 肠胃侦探 (Gut Detective) - 项目背后的故事

> “排查食物过敏和不耐受不应该是一场苦行僧般的修行，而应该是一次跟自己身体谈判的趣味探案。”

## 🌟 项目初衷与愿景

**肠胃侦探** 是一款专为肠易激综合征（IBS）患者及肠胃敏感人群设计的饮食与症状追踪应用。

大多数医疗或健康记录 App 都充斥着冰冷的表格、刺眼的红色警告以及让人感到压迫的“忌口清单”。由于创始人（也就是我）深知这种记录过程的枯燥与心理压力，因此在构建这个项目时，我设定了一个核心目标：**把“得病”的压迫感，转化为“做实验探案”的趣味感。**

这不仅是一个追踪工具，更是一个陪伴你、吐槽你、并用科学方法指导你的“数字福尔摩斯”。

---

## 🎨 UX / UI 设计哲学：克制与温度

项目的 UI 设计没有采用烂大街的渐变色或强烈的医疗蓝，而是走向了 **Soft UI (柔性界面)**：
*   **颜色选择**：以 Ivory (象牙白) 搭配深墨水蓝 (Ink) 为主色调，点缀极少量的柔和警示色（Terra 陶土红、Green-Primary 森林绿）。
*   **去文字化，拥抱 Emoji**：用 🥚、🥛、🍞 代替干瘪的标签词汇。因为在隐秘且不适（比如坐在马桶上）的时候，图形的认知负担远低于文字。
*   **吉祥物 Mascot**：我们引入了一个“肠胃君/侦探”的动态表情系统（从 🫃 进化到 🕵️），它会根据你的打卡状态改变情绪，这提供了情感上的正向反馈。

## 🧪 功能演进里的“后撤步”与纠结

在开发过程中，我们并非一步到位，而是经历了数次重要的**方向调整与自我反思**：

### 1. 从“AI 助手”到“毒舌侦探”的 Persona 重塑
起初，AI 的回复非常刻板（比如：“您好，根据分析，乳糖可能导致了您的腹泻”）。我们觉得太无聊了。
➡️ **改造过程**：我们修改了提示词，要求 AI 拥有“双重人格”——不仅要输出 60 字左右严谨扎实的医学分析卡片，还要针对吉祥物的气泡框输出一句 **15 字以内的一针见血的毒舌吐槽**（比如：“大半夜吃辣条配冰水，肠胃没罢工是它脾气好”）。这种幽默感极大降低了用户的焦虑。

### 2. 实验室模块：从“暴力戒断”到“温和替换”
这也是非常关键的一次迭代。早期版本中，一旦系统发现你对鸡蛋敏感，就会给出一个红色的“戒断”按钮。
➡️ **反思**：但这反人类！食物是营养来源，随意彻底排除某类食物会导致营养不良。
➡️ **重构**：我们重新设计了“实验室”逻辑，将排查分为 Phase 1 和 Phase 2。首选方案永远是 **“温和替换 (Substitution)”**（比如把牛奶换成舒化奶，而不是直接不喝奶），次选才是 **“彻底排除 (Exclusion)”**。我们还加上了**“安全白名单 (免责金牌)”**功能——如果用户明知自己对某个嫌疑食物绝对不过敏，可以直接赦免它，防止 AI 瞎猜。

### 3. 时间线追踪的重构
一开始 AI 只知道你“今天吃了什么”和“昨天吃了什么”。
➡️ **发现问题**：每个人的作息是不同的，“吃得太晚”是一个重要的 IBS 诱因。如果没有时间参考，AI 无法做出准确推断。
➡️ **解决方案**：我们在底层数据结构中精确传入了每一餐的 `[HH:MM]`，并教会了 AI 新的推理规则：不要求绝对“早睡早起”，而是对比“两餐间隔是否过久”、“是否深夜突击高脂食物”、“今日作息与昨日相比的反差”。

## 🏗️ 技术架构

本项目是一个典型的现代化前沿 Web 应用，注重轻量化与快响应：

*   **前端框架**：React 18 + Vite 开发环境。
*   **状态管理**：采用 `Zustand` 替代 Redux。它的 API 极其精简，完美支撑了我们复杂的实验状态追踪和过去十四天历史记录的派生计算。
*   **CSS 工程化**：`TailwindCSS`。借助 Tailwnd 的 Utility-first 特性，我们快速构建了精细的圆角、阴影和过渡动画，同时保持极小的 CSS 体积。
*   **日期处理**：`date-fns` 用于复杂的跨日排查、周期计算及补录日历支持。
*   **AI 智脑引擎**：
    *   底层调用的是 **DeepSeek-V3** 语言模型（通过 SiliconFlow 平台分发），它的推理能力不仅媲美甚至在部分中文语境下超越了顶尖的闭源模型。
    *   **架构退阶与部署挑战**：在将应用推向上线时，我们先后遭遇了 Netlify 配额耗尽、Cloudflare 邮箱验证堵塞等经典的“出海部署阵痛”。为了确保应用不因服务器宕机而瘫痪，我们设计了 **Local Direct Mode (本地直连)** 与 **Serverless Proxy (云端边缘计算函数代发)** 双轨并行的策略。

## 🚀 下一步 (What's Next?)

虽已有出色的基础框架，但这只是 Phase 1。我们的下一个目标是：
1.  **数据可视化图表**：将 14 天的“问题程度”拟合为折线图，更直观展现健康趋势。
2.  **更强的自定义引擎**：除了预设的 12 种核心食物，支持完全自定义追踪（哪怕是患者想追踪“加了防腐剂的薯片”），并由 AI 即时生成该自定义食物的科学替代测试方案。
3.  **成就与游戏化**：连续 7 天无不适打卡，为侦探解锁新的装备。
